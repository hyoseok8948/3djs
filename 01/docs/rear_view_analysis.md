# 핑크색 옷(분홍옷) 인물 추적 및 뒷모습(Back View) 인식 분석

이 문서는 영상 내 특정 색상의 옷(분홍옷)을 입은 사람을 추적하는 방법과, MediaPipe Pose 모델이 뒷모습을 어떻게 추적하는지에 대한 기술적 한계와 원리를 설명합니다.

---

## 1. 핑크색 옷 인물 지정 추적 방법 (How to track the person in Pink)

현재 `App.tsx`에서 구현된 MediaPipe 다중 인물 인식(`numPoses: 5`)은 사람의 **"위치 배열"**만 반환할 뿐, 그 사람이 입은 **"옷의 색상"**은 알려주지 않습니다.
따라서 AI가 "이 사람이 핑크색 옷을 입고 있는지" 스스로 판단해서 해당 인물만 골라내게 하려면 추가적인 비전 컴퓨팅(Computer Vision) 처리가 필요합니다.

### 🟡 방법 A: 고정 인덱스(Index) 또는 위치(Position) 추적 (가장 쉬움)
영상에서 분홍옷을 입은 사람이 주로 화면 중앙을 차지하거나, 처음에 나오는 인원이라면 **UI에서 [P1], [P2] 버튼을 눌러보며 해당 사람이 매핑되는 번호를 찾아 고정**하는 방식입니다.
* 장점: 당장 코드를 수정하지 않아도 됩니다.
* 단점: 해당 인물이 화면 밖으로 나갔다가 다시 들어오거나 다른 사람과 겹치면 인덱스 번호가 바뀔 수 있습니다.

### 🔴 방법 B: 골격 위치 기반 바운딩 영역 색상 추출 (정석 구현)
MediaPipe가 찾아낸 각 사람의 **가슴 영역 좌표(예: 왼쪽/오른쪽 어깨 중앙)**를 계산한 뒤, 원본 비디오 화면(Canvas)에서 해당 좌표 주변의 픽셀 색상값(RGB)을 읽어옵니다. 그 색상이 "분홍색 계열"인 사람만 자동으로 찾아내어 그 랜드마크 데이터만 아바타로 넘기는 필터 필터링 로직을 구축합니다.
* 장점: 사람 위치가 섞여도 무조건 핑크색 옷만 찾아냅니다.
* 단점: 색상 판별 로직(Color Range Detection)을 `detectPose` 루프 내에 추가해야 하므로 연산량이 증가합니다.

> **💡 현재 제시된 가이드:**
> 우선 완성된 화면의 P1 ~ P5 버튼을 눌러보며 핑크색 옷을 입은 사람의 식별 번호를 찾아보세요. 영상에서 대상이 고정적인 위치(예: 센터)에 있다면 번호가 크게 바뀌지 않습니다!

## 2. MediaPipe의 뒷모습(Rear View) 추적 가능성

**"뒷모습도 인식이 가능한가?"에 대한 대답은 "YES"입니다. 인식은 가능합니다. 하지만 제약 사항이 따릅니다.**

Google의 MediaPipe Pose 모델(BlazePose)은 근본적으로 수백만 장의 이미지를 학습하여 관절 위치를 유추하는 딥러닝 모델입니다. 놀랍게도 이 데이터셋 안에는 사람의 앞모습뿐만 아니라 **옆모습, 뒷모습**까지 포함되어 있습니다.

### 뒷모습 인식의 특징과 한계점 (Limitations)

1. **좌우 반전 (Left/Right Swap) 발생 주의**
   MediaPipe는 사람의 형태적 실루엣으로 관절을 유추하기 때문에, 완벽한 뒷모습일 경우 AI가 "이 사람이 앞을 보고 있는지, 뒤를 보고 있는지" 착각하는 경우가 종종 발생합니다. 이 경우 아바타의 **왼팔과 오른팔 움직임이 거꾸로(반전)** 나타나는 현상이 생깁니다.
   * **해결법**: 영상 원본이 뒷모습이라는 것을 알고 있다면, 프론트엔드 코드 내에서 억지로 데이터의 좌우(`11번(왼어깨)` 데이터와 `12번(오른어깨)` 데이터 등)를 Swap 시켜버리는 보정 방식이 존재합니다.

2. **코와 눈 랜드마크(안면부) 유실로 인한 정확도 하락**
   MediaPipe는 0번(코)과 1,2,3번(눈 주변)의 위치를 기준으로 사람의 머리 각도 방향(Quaternion)을 유추합니다. 하지만 뒷모습은 안면 랜드마크가 전혀 보이지 않기 때문에, 목업(Neck) 회전이나 몸통(Spine)의 정확한 3D 비틀림 각도를 계산하는 데 오차가 커질 수 있습니다.

3. **Z축(Depth) 값 불안정 (가장 큰 문제)**
   기존 `pose_data_flow.md`에서 설명했던 앞/뒤 깊이 값(`Z`)이 뒷모습일 때 얕아지거나 튀는 현상이 잦습니다. 팔을 앞으로 뻗었는지 가슴으로 당겼는지 분간하기 어렵기 때문입니다.

### 결론 요약 (Conclusion)
* **뒷모습 추적은 충분히 동작합니다.**
* 하지만 앞모습이나 옆모습 영상보다 **정확도가 다소 떨어질 수 있으며**, 만약 아바타의 좌우 팔다리가 반대로 움직이는 현상(Flip)이 나타난다면 코드 단에서의 수동 보정 스크립트를 추가해야 합니다.
