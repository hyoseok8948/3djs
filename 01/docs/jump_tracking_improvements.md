# 점프(Jump) 동작 추적의 카메라 각도 한계 및 개선 방법

이 문서는 MediaPipe Pose를 이용한 단일 비디오 추적에서 아바타가 **위아래로 점프하는 동작(수직 이동, Vertical Translation)**을 제대로 모방하지 못하는 원인과 이를 코드로 보완할 수 있는 개선 방향을 분석합니다.

---

## 1. 점프 동작이 제대로 구현되지 않는 원인 (Limitations)

### 1) "뼈의 회전각도(IK)"만 계산하는 현재 구조
현재 `Skeleton3D.tsx`의 핵심 로직(`aimBone`)은 각 랜드마크 사이의 **각도(Direction Vector)**만을 계산하여 아바타의 관절을 "비틀어(Rotate)" 줍니다. 즉, 아바타 모델 자체의 **절대적인 중심점(Hips Position) 위치**는 월드 좌표 `Y=0` (바닥 근처)에 거의 고정되어 있고, 팔과 다리만 허우적대도록 세팅되어 있습니다.

### 2) 2D 카메라 원근법(Perspective)에 의한 Z값/Y값의 왜곡
일반적인 2D 비디오(카메라 1대로 찍은 영상)에서 사람이 점프를 하면, 프레임 상에서 사람의 Y 픽셀 값은 위로 올라갑니다.
하지만 MediaPipe는 이 사람이 제자리에서 위로 뛴 것인지, 아니면 앞으로/뒤로(Z축) 이동하면서 카메라와 멀어지거나 가까워지는 바람에 상대적인 크기(Scale)와 픽셀 위치가 변한 것인지 완벽하게 분간해내지 못합니다. 
(예: 카메라 앵글이 살짝 위에서 아래를 향해(High Angle) 찍고 있다면, 점프를 해도 발의 Y값이 생각보다 크게 변하지 않습니다.)

---

## 2. 점프 구현 개선 방법 (Improvement Strategies)

아바타가 실제로 바닥에서 발을 떼고 허공으로 '점프'하게 만들려면, **아바타의 최상위 부모 뼈대(Root Bone 또는 Hips)의 Y축 위치값(Position)**을 영상 속 랜드마크의 Y값 변화량에 맞춰 직접 위아래로 움직여 주는 로직을 추가해야 합니다.

### 방법 1: 골반(Hips) 랜드마크의 Y값을 아바타 전체의 높이(Height)로 연동
가장 간단한 개선 방식입니다. 영상 속 인물의 골반(23, 24번) 또는 발목(27, 28번)의 평균 Y 좌표(상하 높이) 변화량을 계산하여, 아바타 전체를 감싸고 있는 `Group` 또는 아바타의 중심 `Hips Bone`의 `position.y`에 직접 대입해주는 것입니다.

*   **구현 아이디어 (Skeleton3D.tsx 내부 `useFrame` 수정):**
    ```typescript
    // 현재 프레임의 왼쪽/오른쪽 골반(Hip)의 평균 높이(Y)를 구함
    const currentHipY = (lHipV.y + rHipV.y) / 2;
    
    // 첫 번째 프레임일 때, 바닥에 닿아있는 기준점(Baseline) Y값을 저장해둠
    if (!initialHipY.current) initialHipY.current = currentHipY;
    
    // 차이값(Delta) 계산
    const deltaY = currentHipY - initialHipY.current;
    
    // 모델의 최상위 Bone(예: mixamorigHips)의 위치(position)를 변화량만큼 위로 띄움
    const hipsBone = getBone("mixamorigHips");
    if (hipsBone) {
       // 원래 위치 + 변화량 (스케일 보정 필요)
       hipsBone.position.y = REST_HIP_HEIGHT + (deltaY * jumpScaleFactor); 
    }
    ```

### 방법 2: 발끝(Foot Index) 기준 땅바닥(Floor) 판별 오프셋
단순히 골반 Y값만 따라가면, 사람이 카메라 앞으로 다가올 때(크기가 커질 때)도 골반 픽셀 위치가 변해서 아바타가 허공에 붕 뜨는 오류가 발생합니다.
이럴 땐 양 발목(27, 28) 중 **가장 아래쪽에 위치한 발의 Y 좌표**를 항상 바닥(Y=0)으로 간주하고, 나머지 뼈대들의 높이를 상대적으로 계산해주는 코드를 짭니다. 즉, 두 발 중 하나라도 바닥에 닿아있으면 점프가 아니고, 두 발의 가장 아랫부분이 기존 바닥선 위로 크게 떴을 때만 "점프"로 인식하여 아바타를 공중으로 올립니다.

### 방법 3: 가속도(Velocity) 기반 물리 엔징 적용 (고급)
점프의 정점(체공 시간)과 떨어지는 중력 가속도를 자연스럽게 표현하기 위해, 랜드마크의 Y값이 급격히 변하는 순간을 트리거(Trigger)로 잡아 Three.js나 React-Three-Rapier 같은 3D 물리 엔진에 `applyImpulse(위로 점프)`를 줘서 아바타가 포물선을 그리며 스스로 점프하고 바닥에 떨어지도록 시뮬레이션합니다. 

---

## 3. 요약 (Conclusion)

현재의 3D 모션 매핑 시스템은 **제자리에서 팔다리를 비트는 회전(IK)**에는 매우 탁월하지만, **공간을 이동하는 스케일/오프셋(Translation)** 처리가 빠져있어 점프가 되지 않는 것입니다.

만약 댄스 영상이나 점프가 많은 영상을 위해 이 기능이 꼭 필요하시다면, `Skeleton3D.tsx` 내부에 **발끝 최소 Y값을 기준으로 골반뼈(Hips)의 전체 높이를 실시간으로 올렸다 내렸다 하는 "방법 2"**의 로직을 추가해 보는 것을 적극 권장합니다.
