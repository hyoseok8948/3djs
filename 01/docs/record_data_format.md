# 움직임(Motion Tracking) 추출 데이터 포맷 가이드

이 문서는 사용자가 `App.tsx` 화면에서 **"15초 데이터 기록 시작"** 버튼을 눌러 추출한 JSON 파일(`pose_tracking_15sec.json`)의 데이터 구조를 설명합니다.

---

## 1. 개요 (Overview)

앱 하단에 추가된 추출 버튼을 누르면, **버튼을 누른 시점부터 15초 동안** AI 모델이 인식한 아바타의 관절 랜드마크(Landmark) 데이터가 매 프레임(약 30FPS ~ 60FPS)마다 배열 포맷으로 차곡차곡 쌓여, 15초 후 하나의 JSON 파일로 브라우저에 자동 다운로드됩니다.

*   파일명: `pose_tracking_15sec.json`
*   특징: 기존에 3D 아바타가 화면에서 실시간으로 움직이던 좌표(해부학적 좌우 스왑 반영 + 3D 스케일 공간 반영)들이 **시간 순서대로(Time Series)** 그대로 녹화됩니다.

---

## 2. JSON 데이터 구조 (Data Structure)

추출된 파일은 **배열(Array) 형태**의 루트 요소로 시작하며, 배열의 각 요소를 하나의 **프레임(Frame)** 단위로 취급합니다.

```json
[
  // 첫 번째 프레임 데이터
  {
    "time_sec": 0.016, 
    "landmarks": [
      { "x": 0.0, "y": -2.0, "z": 0.1 },   // 0번(코) - 아바타는 사용안함
      // ... 
      { "x": 0.5, "y": 1.2, "z": -0.8 },   // 11번(왼어깨)
      { "x": -0.5, "y": 1.2, "z": -0.8 },  // 12번(오른어깨)
      // ... 28번까지 총 33개의 데이터
    ]
  },
  
  // 두 번째 프레임 데이터
  {
    "time_sec": 0.033, 
    "landmarks": [ ... ]
  },

  // ... 계속 반복되어 15.0초에 도달할 때까지 쌓임
]
```

### 🔹 상세 필드 설명
| 필드명 | 타입 | 설명 |
| :--- | :--- | :--- |
| `time_sec` | `Number` | 기록 시작 버튼을 누른 후 경과된 시간입니다. (단위: 초). 밀리초 단위 세번째 자리까지 표기됩니다. (예: `1.234` ➡️ 1초 234밀리초 경과 지점) |
| `landmarks` | `Array` | 해당 시간(Frame)에 추출된 33개의 3D 공간 랜드마크 위치 좌표 묶음입니다. |
| `landmarks[n].x` | `Number` | n번 관절의 좌우 위치 (MediaPipe 11~16, 23~28 인덱스를 그대로 씁니다) |
| `landmarks[n].y` | `Number` | n번 관절의 상하 위치 |
| `landmarks[n].z` | `Number` | n번 관절의 깊이 위치 (뒷모습이 기준이기 때문에 양수가 화면 뒤쪽을 향함) |

---

## 3. 데이터 활용 방법 (Usage Guide)

여러분이 이 추출된 JSON을 다시 3D 모델이나 다른 플레이어에 밀어 넣으려면 다음과 같이 구현할 수 있습니다.

**기존 `Skeleton3D` 컴포넌트 활용하기:**
`Skeleton3D`는 어차피 "현재 프레임의 `landmarks` 배열 1개"를 입력으로 받습니다. 따라서 이 JSON 파일을 로드한 뒤, 재생 시간에 맞춰 배열을 순회하면서 `Skeleton3D`의 `landmarks` 속성으로 `landmarks[n]` 데이터를 하나씩 전달해주면, 이전에 녹화된 동작이 "리플레이(Replay)"되어 똑같이 재생됩니다.
