# 단일 인물 뒷모습(Rear View) 영상 추적 가능성 분석 (`yellow.mp4`)

이 문서는 제공된 `yellow.mp4` 영상(단 한 명의 사람만 등장하며, 주로 뒷모습을 보임)을 현재의 `Skeleton3D` 시스템에서 추적할 수 있는지에 대한 기술적 분석과 구현 가이드를 제공합니다.

---

## 1. 결론 (Conclusion)

**네, 완벽하게 가능합니다.**

`yellow.mp4` 영상은 이전 영상(`tennis.mp4`)처럼 다중 인물(Multi-Pose)을 골라내는 복잡한 로직이 필요하지 않으므로 구현이 훨씬 간단합니다. 오히려 한 사람만 화면에 꽉 차게 나오는 댄스나 동작 분석 영상이라면, MediaPipe 모델의 인식률(Confidence Score)이 더 높아져서 안정적으로 모션을 캡처할 수 있습니다.

---

## 2. 뒷모습 영상(`yellow.mp4`) 적용 시 장점과 주의점

단 한 명의 뒷모습 영상을 아바타에 적용할 때 어떤 특징이 나타나는지, 그리고 어떤 보정이 필요한지 정리했습니다.

### ✅ 장점 (Advantages)
1. **복잡한 필터링 불필요:** 영상에 사람이 한 명뿐이므로, `numPoses: 1` 설정만으로도 충분합니다. P1, P2 등 대상 전환 버튼 로직을 뺄 수 있어 코드가 깔끔해집니다.
2. **높은 프레임 레이트 및 인식률:** 여러 명을 동시에 연산하지 않기 때문에, 로컬 브라우저 환경에서도 끊김(Lag) 없이 매우 부드럽고 가벼운 실시간 처리가 가능합니다.

### ⚠️ 주의점: "좌우 반전(Left/Right Swap)" (중요)
`rear_view_analysis.md`에서도 언급했듯, MediaPipe는 모델 구조상 **기본적으로 사람의 앞모습을 기준**으로 뼈대에 좌(`Left`) / 우(`Right`) 라벨을 부여합니다.

만약 영상 속 인물의 뒷모습을 그대로 모델에 밀어 넣으면, 영상 속 인물이 (영상 기준) 오른쪽 팔을 들었을 때 아바타도 똑같이 아바타 기준의 (영상 기준) 오른쪽 팔을 들게 됩니다. 
*사용자(카메라) 입장에서는 이는 마치 **거울을 보는 것과 같은 효과(Mirror Effect)**가 됩니다.*

**이것을 "어떻게 보여주고 싶은가?"에 따라 코드 설정이 달라집니다.**

1. **거울 모드 (Mirror Mode) - 기본 거울 동작**
   * 내가 영상의 사람 등을 보면서 따라 하는 느낌 (영상 속 인물이 화면 오른쪽으로 팔을 뻗으면 아바타도 화면 오른쪽으로 팔을 뻗음)
   * **적용 방법:** 현재 `Skeleton3D.tsx`에서 사용 중인 기본 설정입니다. 별도로 수정할 필요가 없습니다.

2. **실제 방향 모드 (Real Direction Mode) - 좌우 데이터 강제 스왑**
   * 해부학적으로 정확히 영상 속 인물의 '오른팔'이 아바타의 '오른팔'과 매칭되길 원할 때. (인물이 화면 오른쪽으로 팔을 뻗으면, 그건 뒷모습이므로 인물의 <u>'오른팔'</u>입니다. 아바타 역시 해부학적 <u>'오른팔'</u>을 뻗어야 함)
   * **적용 방법:** `Skeleton3D.tsx` 내부나, 랜드마크 배열을 넘기기 전(App.tsx)에 MediaPipe의 **왼쪽 뼈대 배열 인덱스(예: 11, 13, 15)와 오른쪽 뼈대 배열 인덱스(예: 12, 14, 16)의 값을 강제로 서로 바꿔치는(Swap)** 데이터 가공 로직이 필요합니다.

## 3. `App.tsx` 적용 가이드 (How to switch video)

`yellow.mp4` 영상을 적용하려면 `App.tsx`의 `<video>` 부분을 다음과 같이 한 줄만 수정하면 됩니다. 사람이 한 명이므로 이전 단계에서 만든 `UI` 컨트롤도 필요 없습니다.

```tsx
// App.tsx
// ...

return (
  // ...
  {isWeb ? (
    <View style={StyleSheet.absoluteFill}>
      <video
        ref={videoRef}
        // 👇 이 부분을 수정합니다
        src={require('./assets/video/yellow.mp4')} 
        playsInline
        muted
        loop
        onLoadedData={handleVideoLoaded}
        style={{ width: '100%', height: '100%', objectFit: 'cover' }}
      />
    </View>
  // ...
)
```

> **💡 다음 스텝 제안:**
> `yellow.mp4`로 영상을 즉시 교체하여 동작을 확인해 보고 싶으시거나, 위에서 언급한 **"좌우 반전(Swap) 로직"**을 적용해 정확히 아바타의 해부학적 방향을 맞추고 싶으시다면 적용해 드릴 수 있습니다.
